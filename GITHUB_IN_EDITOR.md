# Работа с GitHub прямо из редактора (VS Code / Cursor)

## Встроенные возможности редактора

VS Code и Cursor имеют встроенную поддержку Git и GitHub. Вы можете работать с репозиторием прямо из редактора без использования командной строки.

---

## Способ 1: Публикация репозитория через интерфейс редактора

### Шаг 1: Инициализация Git (если еще не сделано)

1. Откройте панель **Source Control** (иконка ветки слева или `Ctrl+Shift+G`)
2. Если Git еще не инициализирован, нажмите **"Initialize Repository"**
3. Или используйте команду: `Ctrl+Shift+P` → `Git: Initialize Repository`

### Шаг 2: Добавление файлов

1. В панели **Source Control** вы увидите все измененные файлы
2. Нажмите **"+"** рядом с файлами или **"Stage All Changes"** для добавления всех файлов
3. Или используйте `Ctrl+Shift+P` → `Git: Stage All Changes`

### Шаг 3: Создание коммита

1. Введите сообщение коммита в поле вверху (например: "Initial commit")
2. Нажмите **"Commit"** (галочка) или `Ctrl+Enter`
3. Или используйте `Ctrl+Shift+P` → `Git: Commit`

### Шаг 4: Публикация в GitHub

1. После коммита появится кнопка **"Publish Branch"** или **"Sync Changes"**
2. Нажмите на нее
3. Выберите **"Publish to GitHub"**
4. Выберите:
   - **Public** (открытый) или **Private** (приватный)
   - Имя репозитория (например: `Abricol_Assistant`)
5. Нажмите **"OK"**

**Готово!** Репозиторий будет создан на GitHub и код загружен автоматически.

---

## Способ 2: Подключение к существующему репозиторию

Если репозиторий уже создан на GitHub:

1. Откройте панель **Source Control** (`Ctrl+Shift+G`)
2. Нажмите на **"..."** (три точки) вверху панели
3. Выберите **"Remote"** → **"Add Remote"**
4. Введите:
   - **Name**: `origin`
   - **URL**: `https://github.com/ваш-username/Abricol_Assistant.git`
5. После коммита нажмите **"Sync Changes"** или **"Push"**

---

## Способ 3: Через Command Palette

1. Нажмите `Ctrl+Shift+P` (или `F1`)
2. Введите команду:
   - `Git: Publish to GitHub` - для публикации нового репозитория
   - `Git: Push` - для отправки изменений
   - `Git: Pull` - для получения изменений
   - `Git: Clone` - для клонирования репозитория

## Визуальные индикаторы в редакторе

### Цветовая индикация файлов:

- **Зеленый** (U) - неотслеживаемый файл (новый)
- **Желтый** (M) - измененный файл
- **Красный** (D) - удаленный файл
- **A** - добавленный файл

### В строке статуса (внизу):

- **Ветка**: показывает текущую ветку (например: `main`)
- **Синхронизация**: показывает статус синхронизации с GitHub
  - `↑` - есть локальные коммиты для отправки
  - `↓` - есть изменения на GitHub для получения
  - `✓` - все синхронизировано

## Основные действия через интерфейс

### Просмотр изменений:

1. Откройте файл в редакторе
2. Изменения будут подсвечены:
   - **Зеленым** - добавленные строки
   - **Красным** - удаленные строки
3. Нажмите на строку для просмотра diff (различий)

### Отмена изменений:

1. В панели **Source Control** найдите файл
2. Правый клик → **"Discard Changes"**
3. Или используйте `Ctrl+Shift+P` → `Git: Discard Changes`

### Создание новой ветки:

1. Нажмите на название ветки внизу (например: `main`)
2. Выберите **"Create new branch"**
3. Введите имя ветки (например: `feature/new-feature`)
4. Или используйте `Ctrl+Shift+P` → `Git: Create Branch`

## Расширения для GitHub (опционально)

Для расширенной работы с GitHub установите расширения:

1. Откройте **Extensions** (`Ctrl+Shift+X`)
2. Найдите и установите:
   - **GitHub Pull Requests and Issues** - для работы с PR и Issues
   - **GitLens** - расширенная визуализация Git истории
   - **Git Graph** - графическое представление веток

---

## Аутентификация в GitHub

При первом использовании GitHub из редактора:

1. Редактор попросит авторизоваться
2. Нажмите **"Sign in with GitHub"**
3. Откроется браузер для авторизации
4. Разрешите доступ редактору к GitHub
5. Или используйте **Personal Access Token**:
   - Создайте токен: GitHub → Settings → Developer settings → Personal access tokens
   - Используйте токен при запросе пароля

---

## Быстрая инструкция (краткая)

1. **Откройте Source Control**: `Ctrl+Shift+G`
2. **Добавьте файлы**: Нажмите **"+"** или **"Stage All Changes"**
3. **Создайте коммит**: Введите сообщение и нажмите **"Commit"**
4. **Опубликуйте**: Нажмите **"Publish Branch"** → Выберите **"Publish to GitHub"**
5. **Готово!** Репозиторий создан и код загружен

## Решение проблем

### Проблема: "Git not found"

**Решение:**
- Установите Git: https://git-scm.com/download/win
- Перезапустите редактор

### Проблема: Кнопка "Publish Branch" не появляется

**Решение:**
1. Убедитесь, что создан хотя бы один коммит
2. Проверьте, что Git инициализирован: `Ctrl+Shift+P` → `Git: Initialize Repository`

### Проблема: Ошибка аутентификации

**Решение:**
1. `Ctrl+Shift+P` → `Git: Sign Out`
2. Попробуйте снова: `Ctrl+Shift+P` → `Git: Sign In`
3. Или используйте Personal Access Token

## Полезные горячие клавиши

- `Ctrl+Shift+G` - открыть панель Source Control
- `Ctrl+Shift+P` - открыть Command Palette
- `Ctrl+Enter` - создать коммит (в панели Source Control)
- `Ctrl+K Ctrl+S` - открыть настройки горячих клавиш

## Сравнение способов

| Способ | Когда использовать |
|--------|-------------------|
| **Интерфейс редактора** | Для начинающих, быстрая публикация |
| **Command Palette** | Для опытных пользователей, больше контроля |
| **Терминал/PowerShell** | Для автоматизации, скриптов, сложных операций |

## Рекомендация

Для первого раза используйте **Способ 1** (через интерфейс редактора) - это самый простой и быстрый способ опубликовать проект в GitHub.
